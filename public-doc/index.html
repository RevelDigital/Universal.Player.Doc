revel-universal-player documentation

RevelDigital Universal Player

Application was built with angular 11, Electron 12, GSAP, and other supporting libraries.

General Information

Built With

  • Anguler 11 - The web framework used
  • Typescript 4.7.0 - Language Version
  • Hammer.js - Gestures Support Library
  • GreenSock.js - Animation Library /

Getting Started

Base Script

Controller.onTemplateInitialized.subscribe(
    () => { /* Triggered when the template is created */
    }
);

Controller.onTemplateTerminated.subscribe(
    () => { /* Triggered when the template is destroyed */
    }
);

Controller.onTemplateClicked.subscribe(
    () => { /* Global click handler for entire template */
    }
);

Gestures Use HammerJS

Implemented Gestures:

  • Pan
  • Pinch
  • Press
  • Rotate
  • Swipe
  • Tap

Implemented Module Types

  • Clock - Analog or digital clock.

  • StaticImage - Single static image.

  • Gadget - Google Gadget.

  • Text - Static text.

  • RichText - Richly formatted text.

  • Image - Image slideshow with transitions.

  • Web - Web content.

  • Ticker - Scrolling marquee/ticker.

  • Shape - An image of a shape

  • QRCode - A QR code of text

  • HotSpot - Touch regions in kiosk applications.

  • TV - Live TV or video feed.

  • Page - A container for other components to give a group of components page behavior

Examples

Listening For A Click

Controller.onClickEvent.subscribe(
    (event: MouseEvent) =>{
        console.log(event);
    });

Listening For A Command

Controller.onCommandCallback.subscribe(
    (command) =>{
        console.log(command);
    }
);

Creating A Custom Animation

GSAP is used for animations. To create a custom animation use the animate method of a module.

The first parameter is the duration of the animation. The second is an TweenMax formatted object (Look at GSAP docs for more info).

 Module.animate(1, {scale:5});

Using An Animation Listener

Module.onAnimation.subscribe(
    (event) =>{
        console.log(event);
    }
);

These Objects can give more control/interface with the BrightSign Player and its hardware.

For cross-platform scripts check if the type is undefined and handle the object appropriately.
 if((typeof new BSDeviceInfo()) !== 'undefined'){
    //your BrightSign Specific Code Here
 }

Notes

  • Global click event is not sent when click occurs within WebZones
  • MarqueeZones are not optimized and can cause performance issues
  • Face analytics may cause heavy performance losses/memory crashes
  • base64 library is dangerous and needs to be replaced
